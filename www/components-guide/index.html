Usage Guide

Component Structure
<!-- Base terminal panel structure --> <terminal-panel title="Optional Title" actions> <div slot="actions"> <!-- Optional action buttons --> </div> <!-- Content goes here --> </terminal-panel>
Content Types
<!-- Command line --> <terminal-line prompt="$" command="command" pipe="|" topic=".topic"></terminal-line> <!-- Output variants --> <div class="terminal-output" data-variant="table"> ASCII table output </div> <div class="terminal-output" data-variant="text"> Simple text output </div> <!-- Event bubbles --> <event-bubble type="goal|notes" content="event content" event-id="event-id"></event-bubble>
Best Practices
  • Always include the terminal-panel wrapper for consistency
  • Use blank headers (no title) when terminal doesn't need a label
  • Combine terminal-line + terminal-output for command + result pairs
  • Use data-variant attributes on output elements to specify content types
  • Event bubbles automatically handle styling based on type attribute
  • Terminal chrome (traffic light dots) are automatically added via CSS
  • Components use Lit Element and import via ES modules from CDN
