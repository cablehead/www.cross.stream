Usage Guide

Component Structure
<!-- Base terminal panel structure --> <div class="terminal-panel"> <div class="terminal-header"> <div class="terminal-title">Title (optional)</div> <div class="terminal-actions"> <!-- Optional action buttons --> </div> </div> <div class="terminal-content"> <!-- Content goes here --> </div> </div>
Content Types
<!-- Command line --> <div class="terminal-line"> <span class="prompt">$</span> <span class="command">command</span> <span class="pipe">|</span> <!-- optional --> <span class="topic">.topic</span> <!-- optional --> </div> <!-- Output variants --> <div class="terminal-output" data-variant="table"> ASCII table output </div> <div class="terminal-output" data-variant="text"> Simple text output </div> <!-- Embedded components --> <div class="embedded-component" data-variant="store"> <div class="event-bubble goal|notes"> <span>event type</span> <span class="event-id">event-id</span> </div> </div>
Best Practices
  • Always include the .terminal-header for consistency
  • Use blank headers (no title) when terminal doesn't need a label
  • Combine .terminal-line + .terminal-output for command + result pairs
  • Use data-variant attributes to specify content types
  • Keep event IDs in .event-id spans for proper styling
  • Terminal chrome (traffic light dots) are automatically added via CSS
